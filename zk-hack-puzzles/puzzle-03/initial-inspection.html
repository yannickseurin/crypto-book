Initial Inspection

First, let us try to define more precisely the goal of the puzzle just from the instructions, without looking into the code yet.

Notation

Given a group G of order q, we will write vectors of scalars and vectors of group elements in bold font, e.g. a=(a0​,…,an−1​)∈(Zq​)n and G=(G0​,…,Gn−1​)∈Gn. For two vectors a=(a0​,…,an−1​)∈(Zq​)n and b=(b0​,…,bn−1​)∈(Zq​)n, their scalar product is defined as ⟨a,b⟩:=i=0∑n−1​ai​bi​modq. Similarly, given a=(a0​,…,an−1​)∈(Zq​)n and G=(G0​,…,Gn−1​)∈Gn, we let ⟨a,G⟩:=i=0∑n−1​ai​Gi​.

Definition of the Proof System

Let us first try to specify the language for which the proof system is designed. We assume that the cyclic group G of prime order q and generators G=(G0​,…,Gn−1​) and H are agreed upon by the prover and the verifier. Then, from the puzzle's description (and also from the code, as we will see), the language L of interest is defined by the relation R={(x,w)∣x=(Ca​,b)∈G×(Zq​)n,w=(a,α)∈Gn×Zq​,Ca​=⟨a,G⟩+αH}.

Recall that the language L defined by this relation consists of all instances x such that there exists a witness w such that (x,w)∈R.

The protocol itself goes as follows:

Proof of Knowledge

One can also check that the proof system is extractable, which follows from a property called special soundness: for any instance in language L, given two accepting transcripts with the same commitments (Cr​,C1​,C2​) but different challenges γ(1) and γ(2), one can compute a witness. Indeed, let ​((Cr​,C1​,C2​),γ(1),(s(1),u(1),t(1))),((Cr​,C1​,C2​),γ(2),(s(2),u(2),t(2)))​ be two accepting transcripts for some instance (Ca​,b) such that γ(1)=γ(2). Let a:=γ(1)−γ(2)γ(1)s(2)−γ(2)s(1)​,α:=γ(1)−γ(2)γ(1)u(2)−γ(2)u(1)​.​ Since both transcripts are accepting, we have ​⟨s(1),G⟩+u(1)H=Ca​+γ(1)Cr​,⟨s(2),G⟩+u(2)H=Ca​+γ(2)Cr​.​ Then ⟨a,G⟩+αH​=γ(1)−γ(2)1​(γ(1)⟨s(2),G⟩−γ(2)⟨s(1),G⟩+γ(1)u(2)H−γ(2)u(1)H)=γ(1)−γ(2)1​(γ(1)(Ca​+γ(2)Cr​−u(2)H)−γ(2)(Ca​+γ(1)Cr​−u(1)H)+γ(1)u(2)H−γ(2)u(1)H)=Ca​.​ Hence, the relation defining the language is satisfied and (a,α) is indeed a witness that the instance (Ca​,b) is in L.

Zero-Knowledge?

The puzzle instructions asks us to find a, which is part of the witness, from a few proofs generated by Bob. If the protocol is zero-knowledge, this shouldn't be possible. Is it zero-knowledge, though, from a theoretical point of view?

The answer is yes. Namely, the interactive protocol as defined above can be shown to be honest-verifier zero-knowledge. The simulator, whose task is to generate a fake transcript distributed as a true transcript between a prover and an honest verifier without knowing the witness (a,α), works as follows:

Something Strange

There is something odd with the way we described the proof system: vector b plays no role in the relation R={(x,w)∣x=(Ca​,b)∈G×(Zq​)n,w=(a,α)∈Gn×Zq​,Ca​=⟨a,G⟩+αH}. In particular, note that the proof that the system is extractable did not use the second equation checked by the verifier (meaning extractability still holds even if this check is omitted).

For this reason, it would make more sense to actually include C1​, the commitment to the inner product ⟨a,b⟩, in the instance and randomness τ in the witness and to define the relation as R′={(σ,ω)∣σ=(Ca​,C1​,b)∈G×G×(Zq​)n,ω=(a,α,τ)∈Gn×Zq​×Zq​,Ca​=⟨a,G⟩+αH,C1​=⟨a,b⟩G1​+τH}.​

By doing this, the protocol would now be specified as follows:

Proverparameters: (G,q,G,H)instance: (Ca​,C1​,b)∈G×G×(Zq​)nwitness: (a,α,τ)∈Gn×Zq​×Zq​relation: Ca​=⟨a,G⟩+αHC1​=⟨a,b⟩G1​+τHr←$​(Zq​)nρ,ν←$​Zq​Cr​:=⟨r,G⟩+ρHC2​:=⟨r,b⟩G1​+νHs:=a+γrmodqu:=α+γρmodqt:=τ+γνmodq​Cr​,C2​​γ​s,u,t​​Verifierparameters: (G,q,G,H)instance: (Ca​,C1​,b)∈G×G×(Zq​)nγ←$​Zq​check that⟨s,G⟩+uH=Ca​+γCr​⟨s,b⟩G1​+tH=C1​+γC2​​​​

However, this is not important for solving the puzzle and we will stick to the original, albeit unnatural, description.
